# AgentSec Workflow Artifact Limitations

AgentSec workflow artifacts are review aids.

They are not:

- verification of a tool;
- security certification;
- quality guarantee;
- product-readiness proof;
- legal compliance proof;
- live gateway protection;
- authorization for external actions.

The public template uses local metadata and repository-provided manifests. Metadata can be incomplete, stale, or wrong. A HOLD/BLOCK result should be reviewed by the repository owner. A pass result means only that the public local checks did not detect the modeled blocking pattern under the supplied metadata.
