# AOI Agent / Operator Dual Positioning

AOI needs two compatible product surfaces.

Agent-facing AOI should feel like fast capability discovery: find useful source-traced candidates quickly, surface missing fields, and reduce wasted tool-selection turns.

Operator-facing AOI should act as pre-use middleware or policy-aware control before tool execution. In advisory mode, agents read route decisions. In enforced mode, operators place AOI before the tool executor.

This dual positioning gives AOI a clearer wedge than a registry alone: agents get speed, operators get receipt-carrying route decisions and escalation paths.

No AOI route decision is security certification, product readiness proof, legal/privacy/license clearance, or external action authorization.
