# GitHub Post Upload Checklist

After a staging upload, inspect the repository manually.

1. Open the GitHub repository.
2. Confirm the owner and repo name are correct.
3. Review `README.md`.
4. Review `docs/public_claim_policy.md`.
5. Review `release/public_beta_v0_2/`.
6. Review `launch/manual_public_beta_v0_2/`.
7. Confirm `data/registry/mcp_registry_public_beta_mcp_dataset_v0_1.json` is present.
8. Confirm no secrets are visible.
9. Confirm no forbidden actions are exposed.
10. Confirm claim boundaries say not verified, not security certified, not a quality guarantee, and not purchasing advice.
11. Decide later whether to make the repository public.
12. Create a GitHub release manually if desired.

Do not claim official standard status, universal adoption, security certification, supplier verification, or quality guarantee.
