# Feedback Reply Claim Boundaries

Feedback reply intake is not:

- reply sending
- GitHub issue creation
- security certification
- code correctness proof
- legal, privacy, license, or eval-clean proof
- product readiness review
- quality guarantee
- no external action authorization

A reply can inform local memory, triage, or a future local second-run candidate. It cannot authorize posting, repo mutation, certification, proof, or live action.
