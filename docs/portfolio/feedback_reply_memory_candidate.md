# Feedback Reply Memory Candidate

A feedback reply memory candidate is a proposed local update to ResidualOps feedback memory.

It can suggest a fixture candidate, evidence request, negative-control candidate, or claim-boundary wording review. It remains `pending_review` unless a future package incorporates it.

It must not contain tokens, private data, private kernels, or claims of certification, proof, product readiness, quality guarantee, or external action authorization.
