# Pilot Dry-Run Operator Checklist

Before running a future pilot dry-run:

1. Confirm the artifacts are local samples or owner-provided local files.
2. Confirm there are no tokens, `.env` content, raw private keys, private kernel values, or unredacted private data.
3. Run the dry-run CLI.
4. Inspect the reviewer readout.
5. Inspect HOLD/BLOCK findings before any second run.
6. Do not post, merge, deploy, upload, train, fetch, clone, call GitHub APIs, call external APIs, run live MCP/tool calls, execute external tools, or use credentials.
7. Do not claim certification, proof, product readiness, quality guarantee, or action authorization.
