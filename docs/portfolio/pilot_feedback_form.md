# Pilot Feedback Form

The pilot feedback form asks for:

- the pilot or dry-run ID;
- the affected vertical;
- the finding that looked wrong or unclear;
- local evidence references;
- reviewer and owner-consent status;
- whether a second local pass is requested.

The form also instructs reviewers not to include tokens, passwords, private keys, `.env` contents, raw PII, private datasets, credentials, private kernel values, provider priors, weights, thresholds, private negative-control seeds, private probe seeds, or commercial routing policy.

The form is not certification, legal/privacy/license/eval-clean proof, code correctness proof, product readiness, quality guarantee, or external action authorization.
