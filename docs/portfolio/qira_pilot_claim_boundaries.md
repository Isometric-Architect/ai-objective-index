# QIRA Pilot Claim Boundaries

A QIRA pilot receipt may say:

- local/offline code-change review artifact
- ALLOW/HOLD/BLOCK findings
- patch scope and risk flags
- behavior-contract gaps
- CI evidence gaps
- known limits and next actions

It must not claim:

- code correctness proof
- security certification
- compliance audit
- quality guarantee
- production readiness
- merge authorization
- deploy authorization
- external repository mutation permission

Private kernel details, scoring calibration values, thresholds, provider priors, anti-gaming policy details, private negative controls, private probe seeds, and commercial routing policy remain non-public.
