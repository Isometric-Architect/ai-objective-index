# QIRA Pilot Receipt Workflow

The ROE-9 workflow is intentionally local:

1. Create a task packet.
2. Classify patch scope and risk flags.
3. Build a behavior contract.
4. Summarize local or copied CI evidence.
5. Create a release-gate receipt.
6. Write a reviewer readout.
7. Store feedback memory for future fixtures, negative controls, and behavior-contract patches.

The workflow records evidence and limits. It does not prove code correctness, certify security, authorize merge, authorize deploy, or mutate an external repository.
