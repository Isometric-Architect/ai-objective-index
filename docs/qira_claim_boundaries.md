# QIRA Claim Boundaries

QIRA public language may say:

- behavior contract
- patch candidate
- validator packet
- residual ledger
- patch receipt
- action license
- scoped local release-gate decision
- ALLOW/HOLD/BLOCK

QIRA public language must not say that a patch is verified, security certified, quality guaranteed, production ready, legally compliant, deployment approved, or authorized for external action.

A test pass is not enough to claim merge readiness. A merge decision is not enough to claim deploy readiness. A local fixture receipt is not independent verification.

QIRA-1 is a productization MVP for local/offline coding governance. It does not run live external repo actions in this package.
