# ResidualOps Distribution Limitations

ResidualOps public distribution artifacts are intentionally conservative.

They can make QIRA, AgentSec, and DataCapsule easier for agents and developers to adopt, but they are not full external execution systems.

## What Distribution Does Not Mean

- not verification
- not security certification
- not quality guarantee
- not production-readiness proof
- not legal review
- not privacy certification
- not license clearance
- not evaluation-clean proof
- not purchasing advice
- not authorization for actions

## Operational Limits

- no active workflow is created by ROE-4
- no GitHub API call is made by ROE-4
- no PR comment is posted
- no live MCP server is called
- no external tool is executed
- no URL is fetched
- no package is uploaded
- no registry metadata is submitted
- no token is requested, printed, stored, or committed

Private calibration, anti-gaming policy, provider priors, negative-control banks, private probe seeds, and commercial routing policy stay outside public distribution artifacts.
