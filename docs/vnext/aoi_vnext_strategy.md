# AOI vNext Strategy

AOI v0.2 remains the public beta engine. It ranks and compares AI tools, APIs, SaaS products, and MCP servers with source traces, missing fields, and decision receipts.

AOI vNext is a strategy and schema upgrade. The positioning shifts from "AI Objective Index / AI search engine" to:

**AI Agent Capability Trust Router**

AOI is not another registry. AOI is the trust and routing layer above registries: it routes AI agents from objectives to capabilities with evidence, residual risk, and usage boundaries.

AOI does not certify security or quality. It does not verify suppliers. It does not provide purchasing advice. It does not perform payment, booking, login, email, purchase, form submission, contract signing, account connection, supplier claim, or profile modification.

The first vNext vertical is **Coding Agent Tool Selection**.

PyPI upload and MCP Registry submission are paused only until vNext alignment is complete.

Package 9B adds the first local Capability Trust Schema MVP. It can build offline route-readiness cards from existing AOI objects and source traces, but it still does not run probes, execute a gateway, certify security, guarantee quality, or assert product readiness.

Packages 9C-9E add read-only Objective Router REST/MCP surfaces, local ExecutionReceipt memory, and local-only Probe-before-Use. Together they form a minimum router loop, not a security gateway or external action authorization system.

Package 9F gates distribution so PyPI/TestPyPI and MCP Registry work can resume only after README, package version, `.mcp/server.json`, OpenAPI, MCP manifest, docs, and claim boundaries are aligned.
